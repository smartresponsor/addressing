Address apply guide (opt-in)

Goal

- Apply a patch generated by tooling (e.g., Codex) in a controlled way.

Rules

- Never apply automatically.
- Always require SR_ALLOW_APPLY=1.
- Prefer: plan -> codex-review -> generate patch -> apply -> canon-check -> tests.

Patch format

- Unified diff (git apply compatible).
- Paths must be relative to repo root.

Recommended workflow (PowerShell)

1) Ensure clean working tree:
   git status

2) Apply patch:
   $env:SR_ALLOW_APPLY = "1"
   .\Domain\Tool\ai-apply.ps1 -Patch report\address-change.patch -TestCmd "composer test"

3) Commit:
   git add -A
   git commit -m "address: apply canon patch"
